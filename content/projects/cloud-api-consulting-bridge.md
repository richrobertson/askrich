## Recruiter Summary

This period bridged earlier enterprise consulting work into later platform roles by combining Microsoft cloud/data pipeline delivery with customer-facing API work for Starbucks licensee systems.

## Problem

The work sat at the transition point between classic enterprise consulting and later platform ownership. It required moving comfortably across cloud tooling, data workflows, and business-critical APIs used in real operating environments.

## Scope

- Azure pipeline and data-processing support for Microsoft internal usage-metrics workflows
- REST API delivery for Starbucks licensee rewards earning and redemption
- Client-facing consulting across cloud services and service integrations

## My Role

At Slalom, I provided technical leadership on cloud and API projects for clients including Microsoft and Starbucks, helping deliver both internal data-platform automation and external-facing service integrations.

## Key Design Decisions

- Built Azure Pipelines and Cosmos Scope scripts for Microsoft's internal big-data analysis environment processing Azure usage metrics.
- Delivered REST APIs used by thousands of Starbucks licensee locations to support customer rewards earning and redemption.
- Worked pragmatically across cloud tooling, API contracts, and client-specific operational constraints.

## Technologies

Microsoft Azure, Cosmos Scope, ASP.NET web services, Apigee proxies, REST APIs, Microsoft SQL Server.

## Outcomes

- Strengthened the transition from enterprise delivery tooling into cloud service and API work.
- Delivered practical cloud and API solutions across multiple client contexts.
- Established direct continuity into later Starbucks backend platform leadership.

## Leadership Signals

- Adapted quickly across different client domains and delivery modes.
- Balanced hands-on implementation with technical guidance.
- Showed clear progression from consulting execution into product-platform engineering.
