# Content Authoring Guide

## Purpose

This guide defines how to write high-signal content for retrieval quality.
The objective is recruiter-usable evidence, not marketing copy.

## Required frontmatter

All content files should include YAML frontmatter with at least:
- `id`
- `title`
- `type`
- `summary`
- `tags`
- `updated`

Project briefs should also include `role` and `technologies`.

## Writing project briefs

Use this section order where applicable:

1. **Recruiter Summary**
2. **Problem**
3. **Scope**
4. **My Role**
5. **Key Design Decisions**
6. **Technologies**
7. **Outcomes**
8. **Leadership Signals**

## Retrieval-quality writing principles

- Write concrete claims with context.
- Prefer specifics over broad adjectives.
- Include constraints, tradeoffs, and measurable outcomes.
- Keep terminology consistent across docs.
- Use meaningful headings for chunk-friendly segmentation.

## Avoid weak/fluffy content

Avoid:
- “world-class”, “best-in-class”, “cutting-edge” without evidence,
- vague claims without scope or result,
- dense paragraphs that hide actionable details.

Prefer:
- crisp statements of responsibility,
- explicit impact and technical decisions,
- concise language recruiters can scan quickly.
