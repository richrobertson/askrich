## Recruiter Summary

Led architecture and backend delivery work that helped Starbucks migrate My Starbucks Rewards for the U.S. and Canada from Oracle-backed systems to the Unified Commerce Platform without disrupting customer-critical workflows.

## Problem

Rewards systems were central to the Starbucks customer experience, so migration work had to modernize platform boundaries and backend integrations without introducing visible customer disruption.

## Scope

- Rewards platform migration support for U.S. and Canada
- Backend API and service architecture across account, loyalty, and commerce-adjacent flows
- Cross-team technical direction and implementation standards

## My Role

As Engineer Lead, I guided technical direction and execution across backend API and platform teams supporting the Starbucks app used by more than 17 million active customers.

## Key Design Decisions

- Drove architecture changes that enabled migration of My Starbucks Rewards from Oracle-based systems to the Unified Commerce Platform.
- Set implementation standards and engineering practices across Account, Loyalty, Mobile Order and Pay, and Payments teams.
- Balanced migration progress with customer-facing reliability requirements in a high-scale consumer environment.

## Technologies

Backend APIs, Java services, loyalty platform integrations, migration-oriented service architecture, customer account systems.

## Outcomes

- Helped enable a seamless rewards platform migration across the U.S. and Canada.
- Preserved reliability for customer-critical loyalty workflows while platform changes were underway.
- Strengthened architectural consistency across multiple backend teams.

## Leadership Signals

- Provided technical direction across multiple teams rather than a single service surface.
- Balanced platform modernization with customer-experience risk.
- Operated effectively in a large-scale consumer system with high visibility and real business impact.
